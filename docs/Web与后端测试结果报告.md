# Web与后端测试结果报告

## 1. 测试目标

本轮测试面向 PAL_4G 项目的当前可交付版本，覆盖以下范围：

- 后端接口与核心业务链路
- Web 前端关键页面与交互可用性
- Web 前端关键文案与布局回归
- 前后端联调后的基础功能可访问性

本轮测试组合：

- 后端：`pytest`
- 前端单元测试：`Vitest`
- 前端端到端测试：`Playwright`

## 2. 测试环境

- 后端目录：`backend/`
- 前端目录：`web/`
- Web 本地开发地址：`http://127.0.0.1:5173`
- 后端本地接口地址：`http://127.0.0.1:8001`
- 默认管理员账号：`admin / admin123456`

## 3. 测试范围

### 3.1 后端 `pytest`

覆盖方向：

- 登录鉴权
- 用户管理
- 设备管理
- 报警记录
- 联动指令
- WebSocket 实时事件
- 设备状态历史
- 分页接口

### 3.2 前端 `Vitest`

覆盖方向：

- 登录页交付态文案回归
- 顶部欢迎卡片回归
- 去除技术术语与阶段性文案的可见性校验

### 3.3 前端 `Playwright`

覆盖方向：

- 登录流程
- 仪表盘打开
- 设备管理页可访问
- 用户管理页可访问
- 报警页分页跳页控件可见
- `manager` 角色权限隔离

## 4. 测试结果

本轮实际执行结果如下：

- 后端 `pytest`：`55/55 passed`
- 前端 `Vitest`：`2/2 passed`
- 前端 `Playwright`：`4/4 passed`

实际执行命令：

- 后端：`D:\uv\venvs\pal_4g\Scripts\python.exe -m pytest -q`
- 前端单元测试：`npm.cmd run test`
- 前端端到端测试：`npm.cmd run test:e2e`

本轮 E2E 实际覆盖：

- 登录进入仪表盘
- 超级管理员访问设备管理页
- 超级管理员访问用户管理页
- 报警列表分页跳页控件可见
- `manager` 访问自己名下设备
- `manager` 访问管理员页面被拦截

为保证页面存在真实业务数据，Playwright 测试运行时已通过后端接口插入少量测试数据：

- 1 个测试设备
- 1 条测试报警记录
- 1 个测试经理账号
- 1 个经理名下测试设备

这些测试数据当前未清理，按当前要求保留，后续可统一删除。

## 5. 结果说明

若三类测试均通过，则说明当前版本满足以下基本要求：

- 后端核心接口可正常工作
- Web 关键业务页面可访问
- 登录与后台主导航可正常使用
- 当前交付态文案未再暴露 `JWT`、`MVP`、`二期` 等研发阶段术语
- 分页组件支持直接跳页
- Web 与后端本地联调链路可用

## 6. 当前限制

- Playwright 依赖本地浏览器驱动与前后端服务正常启动
- 若后端未启动在 `8001`，前端联调与 E2E 测试会失败
- 若前端未启动在 `5173`，Web 端 E2E 测试会失败
- 当前测试脚本默认使用管理员账号 `admin / admin123456`

## 7. 测试脚本位置

本轮测试脚本使用本地忽略目录保存，不进入 Git 版本管理：

- `web/tests/vitest/`
- `web/tests/playwright/`
- `web/tests/playwright.config.ts`

测试产物也已忽略：

- `web/playwright-report/`
- `web/test-results/`

## 8. 运行前提

后续复跑测试时，仍需满足以下条件：

- 后端服务运行在 `http://127.0.0.1:8001`
- 前端服务运行在 `http://127.0.0.1:5173`
- 本机已安装：
  - `Vitest`
  - `@playwright/test`
  - `Playwright Chromium`

## 9. 需求符合度结论

基于当前代码、测试结果和需求文档，本项目在不计小程序的前提下，已经达到“可交付前联调与验收”状态，但还不能视为严格意义上的 100% 全量闭环。

当前已满足或基本满足：

- 账号密码登录与角色区分
- 超级管理员与普通管理者的权限隔离
- 设备管理、设备绑定、设备移除
- 仪表盘总览、报警记录、手动控制
- 后端设备接入基础能力、报警联动、日志与状态历史
- Web 前端主要业务页面可访问且主流程可用

当前仍需在交付前重点确认：

- 真实设备协议与有人云 / WH-GM5 实际联调是否完成
- 原始需求文档中的“模块”表述，与当前已确认的“一个设备一个 SN、按管理者整套联动”的最新业务口径是否形成正式确认
- 小程序端尚未纳入本轮测试
