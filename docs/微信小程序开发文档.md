# 微信小程序开发文档

## 一、文档说明

- 项目名称：4G 压力报警联动控制系统
- 文档用途：明确微信小程序首期开发范围、技术方案、接口边界、开发阶段和验收标准
- 当前更新时间：2026-04-05
- 对应目录：`miniprogram/`

## 二、当前现状

截至当前版本，项目的移动端开发条件如下：

- 后端已具备小程序首期大部分业务接口
- 已提供账号密码登录、首页摘要、我的设备、报警列表、设备详情、继电器控制、设备绑定与解绑能力
- 已提供 WebSocket 实时事件通道：`/api/v1/ws/events`
- 微信授权登录、账号绑定、订阅消息推送后端链路暂未正式实现
- 仓库中已建立 `miniprogram/` 工程骨架，可直接继续开发

## 三、首期开发目标

微信小程序作为移动端主入口，首期交付以下能力：

1. 账号密码登录
2. 首页摘要与最近报警
3. 我的设备列表
4. 设备详情与模块状态查看
5. 继电器远程控制
6. SN 绑定设备与移除设备
7. 历史报警分页查询
8. 消息订阅授权入口预留
9. 微信授权登录入口预留
10. 修改密码与退出登录

首期不纳入范围：

- 超级管理员后台能力
- 日志中心
- 调度任务
- MQTT 运维状态页
- 文件导出
- 多语言国际化
- 深色模式
- 复杂统计大屏
- 小程序内协议配置管理

## 四、技术方案

### 4.1 技术栈

- 框架：UniApp
- 语法：Vue 3
- 状态管理：轻量级组合式单例 store
- 网络请求：`uni.request`
- 实时通信：`uni.connectSocket`
- 推送入口：微信订阅消息授权接口预留

说明：

- 由于当前仓库内未集成 `uView Plus` 与 `Pinia` 依赖，首期实现采用 UniApp 原生组件与组合式状态模块完成主流程，后续如统一依赖体系后可平滑替换。

### 4.2 目录结构

```text
miniprogram/
├─ api/                # 认证、首页、设备、控制、用户接口封装
├─ components/         # 通用卡片与空态组件
├─ composables/        # WebSocket 实时连接
├─ pages/
│  ├─ login/
│  ├─ home/
│  ├─ devices/
│  ├─ device-detail/
│  ├─ device-bind/
│  ├─ alarms/
│  └─ profile/
├─ stores/             # 登录态单例 store
├─ utils/              # 配置、格式化、鉴权守卫、错误处理、订阅状态
├─ App.vue
├─ main.js
└─ pages.json
```

### 4.3 交互约定

- 所有接口统一携带 Bearer Token
- 401 统一清空登录态并跳转登录页
- 首页和设备详情页同时支持 WebSocket 和轮询兜底
- 控制类操作必须二次确认
- 所有页面统一具备加载态、空态、错误态
- 小程序只面向普通设备账户，不承载管理员菜单

## 五、页面与接口设计

### 5.1 登录页

功能：

- 账号密码直接登录
- 微信授权登录入口预留
- 登录成功后进入首页

已接接口：

- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`

待后续联调：

- `POST /api/v1/auth/wechat-login`
- `POST /api/v1/auth/wechat-bind`

### 5.2 首页

功能：

- 展示设备数、在线设备数、当前告警数、待处理指令数
- 展示最近报警
- 展示我的设备摘要
- 提供绑定设备、进入设备列表、报警列表、我的页面入口
- 提供消息订阅授权入口

已接接口：

- `GET /api/v1/dashboard/my/home`
- `GET /api/v1/dashboard/my/devices`
- `GET /api/v1/dashboard/my/alarms`
- `GET /api/v1/ws/events`

### 5.3 我的设备页

功能：

- 查看当前账号下所有设备
- 支持本地搜索
- 跳转设备详情
- 进入绑定设备页

已接接口：

- `GET /api/v1/dashboard/my/devices`
- `GET /api/v1/ws/events`

### 5.4 设备详情页

功能：

- 展示设备基本信息
- 展示模块在线状态、电量、电压、继电器状态、最后在线时间
- 查看最近报警和最近命令
- 支持远程控制继电器开/关
- 支持从当前账号中移除设备

已接接口：

- `GET /api/v1/devices/{device_id}`
- `GET /api/v1/dashboard/devices/{device_id}`
- `POST /api/v1/relay-commands`
- `POST /api/v1/devices/{device_id}/unbind`
- `GET /api/v1/ws/events`

### 5.5 绑定设备页

功能：

- 输入 SN 绑定设备
- 预留扫码填充 SN

已接接口：

- `POST /api/v1/devices/bind`

### 5.6 历史报警页

功能：

- 分页查看报警记录
- 支持关键词、报警类型、起止日期筛选
- 支持下拉刷新和加载更多

已接接口：

- `GET /api/v1/dashboard/alarms/page`

说明：

- 虽然仓库中存在 `/dashboard/my/alarms/page`，但该接口当前不支持筛选条件，因此首期小程序为满足移动端筛选需求，直接复用已按当前用户权限收敛的 `/dashboard/alarms/page`。

### 5.7 我的页

功能：

- 查看当前账号信息
- 查看消息订阅状态
- 修改密码
- 退出登录

已接接口：

- `POST /api/v1/users/me/change-password`

- 登录成功、失败、失效退出
- 首页摘要准确
- 我的设备列表可正常展示

- WebSocket 断开后页面仍可通过轮询刷新
- 后端异常、弱网、超时场景有明确提示
- 频繁进入退出详情页不出现重复订阅

### 验收标准

- 登录、查看设备、查看报警、控制继电器、绑定设备、修改密码六条链路可跑通
